A.1 SYSTEM INTRODUCTION
================================================================================

The QA Brains E-Commerce Web Application is a practice e-commerce platform
designed for quality assurance testing purposes. It provides a complete online
shopping experience, including user authentication, product browsing, product
detail viewing, cart management, and a multi-step checkout process.

Base URL: https://practice.qabrains.com/ecommerce

The application consists of 7 distinct pages, each serving a specific purpose
in the user journey from login to order completion.

A.2.2 DETAILED FLOW DESCRIPTIONS
--------------------------------------------------------------------------------

FLOW 1: USER AUTHENTICATION
  1. User navigates to https://practice.qabrains.com/ecommerce/login
  2. User enters email and password
  3. User clicks "Login" button
  4. System validates credentials
  5. On success: User is redirected to Home Page
  6. On failure: Error message displayed, user remains on Login Page

FLOW 2: PRODUCT BROWSING
  1. User lands on Home Page after login
  2. User sees all products in a grid layout
  3. User can sort products using the sorting dropdown:
     a. Search — filter by keyword
     b. A to Z — alphabetical ascending
     c. Z to A — alphabetical descending
     d. Low to High Price — price ascending
     e. High to Low Price — price descending
  4. User can click favourite button on any product card
  5. User can click "Add to Cart" on any product card
  6. User can click on a product to view its details

FLOW 3: PRODUCT DETAIL VIEWING
  1. User clicks a product on the Home Page
  2. User is navigated to Product Details Page
     (URL: /product-details?id={product_id})
  3. User sees product image, name, price
  4. User can toggle favourite status
  5. User can adjust quantity using + and - buttons
  6. User can click "Add to Cart"
  7. User can click "Back" button to return to Home Page

FLOW 4: CART MANAGEMENT
  1. User clicks cart icon in header (from any page)
  2. User is navigated to Cart Page
  3. User sees all items with image, name, quantity, price
  4. User can adjust quantity per item using + and - buttons
  5. User can remove items using the remove button
  6. User sees dynamically calculated total price
  7. User can click "Continue Shopping" to return to Home Page
  8. User can click "Checkout" to proceed to checkout flow

FLOW 5: CHECKOUT — INFORMATION ENTRY
  1. User clicks "Checkout" on Cart Page
  2. User is navigated to Checkout: Your Information Page
  3. User enters: Email, First Name, Last Name, Zip Code
  4. User can click "Cancel" to return to Cart Page
  5. User can click "Continue" to proceed
  6. System validates all fields before proceeding
  7. On validation success: User proceeds to Overview Page
  8. On validation failure: Error messages displayed

FLOW 6: CHECKOUT — ORDER REVIEW
  1. User arrives at Checkout: Overview Page
  2. User reviews all order details (read-only):
     a. Product details (image, name, quantity, price, total price)
     b. Payment information
     c. Shipping information
     d. Price total (item total, tax, grand total)
  3. User can click "Cancel" to abort checkout
  4. User can click "Finish" to place the order

FLOW 7: ORDER COMPLETION
  1. User clicks "Finish" on Overview Page
  2. System processes the order
  3. User is redirected to Checkout: Complete Page
  4. User sees success confirmation:
     a. "Checkout: Complete" heading
     b. Green tick icon
     c. "Thank you for your order" message
     d. Dispatch description message
  5. User's cart is cleared
  6. User can click "Continue Shopping" to return to Home Page

FLOW 8: PROFILE ACTIONS (FROM ANY PAGE WITH HEADER)
  1. User clicks Profile dropdown in header
  2. Dropdown shows: Favourites, Logout
  3. Favourites: Navigates to Favourites Page
  4. Logout: Clears session, redirects to Login Page

FLOW 9: LOGOUT
  1. User clicks Profile dropdown → Logout
  2. System invalidates session/token
  3. System clears client-side stored data
  4. User is redirected to Login Page
  5. Browser back button does not grant re-access

A.6 GLOBAL NON-FUNCTIONAL REQUIREMENTS
================================================================================

These requirements apply across ALL pages of the application.

A.6.1 PERFORMANCE
  NFR-G-01  All pages shall load completely within 3 seconds on a standard
            broadband connection (>= 5 Mbps).
  NFR-G-02  API responses shall be received within 5 seconds under normal
            server load.
  NFR-G-03  UI interactions (clicks, hovers, toggles) shall respond within
            100 milliseconds.
  NFR-G-04  No page shall have a Largest Contentful Paint (LCP) greater
            than 2.5 seconds.
  NFR-G-05  No page shall have a Cumulative Layout Shift (CLS) greater
            than 0.1.

A.6.2 SECURITY
  NFR-G-06  All data transmission shall use HTTPS/TLS encryption.
  NFR-G-07  All pages shall be protected against XSS — user inputs shall
            be sanitized.
  NFR-G-08  All pages shall be protected against CSRF using tokens or
            equivalent mechanisms.
  NFR-G-09  All pages shall be protected against SQL Injection.
  NFR-G-10  Authentication tokens shall be stored securely (HTTP-only
            cookies or secure storage).
  NFR-G-11  Error messages shall not reveal sensitive system information.
  NFR-G-12  Rate limiting shall be implemented on critical endpoints.
  NFR-G-13  Session timeout shall be enforced after a period of
            inactivity.

A.6.3 COMPATIBILITY
  NFR-G-14  All pages shall be fully functional on: Google Chrome (latest),
            Mozilla Firefox (latest), Apple Safari (latest), Microsoft
            Edge (latest).
  NFR-G-15  All pages shall function on: Windows 10/11, macOS, Ubuntu
            Linux, Android (mobile browser), iOS (Safari).

A.6.4 RESPONSIVENESS
  NFR-G-16  All pages shall be responsive across: Desktop (>=1024px),
            Tablet (768px-1023px), Mobile (320px-767px).
  NFR-G-17  No page shall require horizontal scrolling on any supported
            viewport.

A.6.5 ACCESSIBILITY
  NFR-G-18  All pages shall conform to WCAG 2.1 Level AA.
  NFR-G-19  All interactive elements shall be keyboard navigable.
  NFR-G-20  All images shall have meaningful alt text.
  NFR-G-21  All form fields shall have associated <label> elements.
  NFR-G-22  Color contrast ratios: minimum 4.5:1 for normal text, 3:1
            for large text.
  NFR-G-23  ARIA attributes shall be used where appropriate.
  NFR-G-24  Screen readers shall navigate all pages in meaningful order.
  NFR-G-25  All interactive elements shall have minimum touch target
            size of 44x44px.

A.6.6 RELIABILITY & AVAILABILITY
  NFR-G-26  The application shall have 99.9% uptime (excluding scheduled
            maintenance).
  NFR-G-27  In case of backend unavailability, pages shall display
            user-friendly error messages.

B.2 FUNCTIONAL REQUIREMENTS
================================================================================

FR-LGN-01: Logo / Home Button
--------------------------------------------------------------------------------
  FR-LGN-01.1   The logo shall be displayed at a prominent position on
                 the login page (top-left or top-center of the header).
  FR-LGN-01.2   The logo shall be rendered as a visible image (PNG, SVG,
                 or equivalent).
  FR-LGN-01.3   The logo shall be clickable (wrapped in an <a> tag or
                 equivalent).
  FR-LGN-01.4   When clicked, the logo shall navigate to the application
                 home page.
  FR-LGN-01.5   The logo shall have an alt attribute with meaningful text
                 (e.g., "QA Brains E-Commerce Home").
  FR-LGN-01.6   On hover, cursor shall change to pointer.
  FR-LGN-01.7   The logo shall maintain aspect ratio across all screen
                 sizes.

FR-LGN-02: "Login" Heading Text
--------------------------------------------------------------------------------
  FR-LGN-02.1   The page shall display the text "Login" as a heading.
  FR-LGN-02.2   The heading shall use an appropriate HTML heading tag
                 (e.g., <h1>, <h2>).
  FR-LGN-02.3   The heading shall be visually prominent (larger font,
                 bold weight).
  FR-LGN-02.4   The heading text shall be static and non-editable.
  FR-LGN-02.5   The heading shall be positioned above the login form
                 fields.

FR-LGN-03: "Email" Label Text
--------------------------------------------------------------------------------
  FR-LGN-03.1   The label "Email" shall be displayed adjacent to the
                 email input field.
  FR-LGN-03.2   The label shall use a <label> HTML element.
  FR-LGN-03.3   The <label> shall be associated with the email input
                 via the "for" attribute.
  FR-LGN-03.4   Clicking the label shall focus the email input field.
  FR-LGN-03.5   The label shall be static, non-editable, and legible.

FR-LGN-04: Email Input Field
--------------------------------------------------------------------------------
  FR-LGN-04.1   The email field shall be an <input> with type="email"
                 or type="text".
  FR-LGN-04.2   The field shall accept alphanumeric and email-valid
                 special characters (@, ., _, -, +).
  FR-LGN-04.3   The field shall have placeholder text (e.g., "Enter
                 your email").
  FR-LGN-04.4   The field shall be initially empty on page load.
  FR-LGN-04.5   The field shall allow type, paste, cut, select all,
                 and delete operations.
  FR-LGN-04.6   The field shall be single-line.
  FR-LGN-04.7   The field shall have a visible border/outline.
  FR-LGN-04.8   When focused, a visual focus indicator shall appear.
  FR-LGN-04.9   The field shall support Tab / Shift+Tab navigation.
  FR-LGN-04.10  Maximum length: 254 characters (RFC 5321).
  FR-LGN-04.11  The field shall be required.

FR-LGN-05: "Password" Label Text
--------------------------------------------------------------------------------
  FR-LGN-05.1   The label "Password" shall be displayed adjacent to the
                 password input field.
  FR-LGN-05.2   The label shall use a <label> HTML element.
  FR-LGN-05.3   The <label> shall be associated with the password input
                 via the "for" attribute.
  FR-LGN-05.4   Clicking the label shall focus the password input field.
  FR-LGN-05.5   The label shall be static, non-editable, and legible.

FR-LGN-06: Password Input Field
--------------------------------------------------------------------------------
  FR-LGN-06.1   The password field shall be an <input> with
                 type="password" by default.
  FR-LGN-06.2   Characters shall be masked by default (dots or
                 asterisks).
  FR-LGN-06.3   The field shall accept alphanumeric, special characters,
                 and spaces.
  FR-LGN-06.4   The field shall have placeholder text (e.g., "Enter
                 your password").
  FR-LGN-06.5   The field shall be initially empty on page load.
  FR-LGN-06.6   The field shall allow type, paste, cut, select all,
                 and delete operations.
  FR-LGN-06.7   The field shall be single-line.
  FR-LGN-06.8   The field shall have a visible border/outline.
  FR-LGN-06.9   When focused, a visual focus indicator shall appear.
  FR-LGN-06.10  The field shall support Tab navigation.
  FR-LGN-06.11  The field shall be required.
  FR-LGN-06.12  Maximum length: 128 characters.

FR-LGN-07: Password Hide/Show Toggle Button
--------------------------------------------------------------------------------
  FR-LGN-07.1   A toggle button shall be positioned inside or adjacent
                 to the password field (right side).
  FR-LGN-07.2   Default State (Hidden): type="password", icon shows
                 "eye with slash" or "closed eye".
  FR-LGN-07.3   Toggled State (Visible): type="text", password shown
                 in plain text, icon shows "open eye".
  FR-LGN-07.4   Clicking again reverts to hidden state.
  FR-LGN-07.5   Toggle shall NOT clear, alter, or reset password
                 content.
  FR-LGN-07.6   Cursor: pointer on hover.
  FR-LGN-07.7   Keyboard accessible (focusable, Enter/Space to
                 activate).
  FR-LGN-07.8   aria-label shall update dynamically ("Show password" /
                 "Hide password").
  FR-LGN-07.9   Toggle shall work correctly regardless of click count.
  FR-LGN-07.10  Default on page load: always hidden (password masked).

FR-LGN-08: Login Button
--------------------------------------------------------------------------------
  FR-LGN-08.1   The button shall display the text "Login".
  FR-LGN-08.2   The button shall be a <button> with type="submit" or
                 equivalent.
  FR-LGN-08.3   Positioned below the password field as primary CTA.
  FR-LGN-08.4   Visually distinct (solid background, contrasting text).
  FR-LGN-08.5   Hover: visual feedback (color change, shadow).
  FR-LGN-08.6   Click (valid inputs): triggers form submission / API
                 call.
  FR-LGN-08.7   Clickable via mouse, Enter key, and touch.
  FR-LGN-08.8   Loading state during API call (spinner, disabled).
  FR-LGN-08.9   Re-enabled after server response.
  FR-LGN-08.10  Keyboard-focusable and accessible.

FR-LGN-09: Login Form Submission & Authentication
--------------------------------------------------------------------------------
  FR-LGN-09.1   Client-side validation on both fields before API call.
  FR-LGN-09.2   POST request to authentication API with email and
                 password.
  FR-LGN-09.3   Password transmitted over HTTPS, never in URL.
  FR-LGN-09.4   On success: redirect to Home Page.
  FR-LGN-09.5   On success: store auth token securely.
  FR-LGN-09.6   On failure (invalid credentials): display error on page.
  FR-LGN-09.7   On server error (5xx): display generic error message.
  FR-LGN-09.8   Error shall NOT indicate whether email or password was
                 wrong (prevent credential enumeration).
  FR-LGN-09.9   Pressing Enter in any field triggers form submission.


B.3 VALIDATION & ERROR HANDLING
================================================================================

B.3.1 Client-Side Validation
--------------------------------------------------------------------------------

  VAL-LGN-01  Email: Field must not be empty.
              Error: "Email is required."

  VAL-LGN-02  Email: Must be valid format (user@domain.com, contains @).
              Error: "Please enter a valid email address."

  VAL-LGN-03  Email: Must not exceed 254 characters.
              Error: "Email is too long."

  VAL-LGN-04  Email: No leading/trailing spaces (auto-trim or reject).
              Error: "Email should not contain leading/trailing spaces."

  VAL-LGN-05  Password: Field must not be empty.
              Error: "Password is required."

  VAL-LGN-06  Password: Minimum length (1 or defined minimum, e.g., 6).
              Error: "Password must be at least [X] characters."

  VAL-LGN-07  Password: Must not exceed 128 characters.
              Error: "Password is too long."

B.3.2 Server-Side Error Responses

  POST     /api/auth/logout            Profile Dropdown   End session

  PATCH    /api/cart/{itemId}          Cart Page          Partial update

  GET      /api/favourites            Favourites Page     Fetch favourites
  POST     /api/checkout/info         Checkout Info       Submit user info
  GET      /api/checkout/overview     Checkout Overview   Fetch order summary
  POST     /api/checkout/complete     Checkout Overview   Place order


J.4 HTTP STATUS CODE REFERENCE
================================================================================

  Status   Meaning                    Usage Context
  ------   -------------------------  ----------------------------------
  200      OK                         Successful GET, PUT, PATCH, DELETE
  201      Created                    Successful POST (new resource)
  204      No Content                 Successful DELETE (no body)
  400      Bad Request                Malformed request, empty cart
  401      Unauthorized               Invalid/expired auth token,
                                      invalid login credentials
  403      Forbidden                  Insufficient permissions
  404      Not Found                  Product not found, invalid ID
  409      Conflict                   Duplicate order attempt
  422      Unprocessable Entity       Validation errors (field-level)
  429      Too Many Requests          Rate limit exceeded (login)
  500      Internal Server Error      Server failure
  502      Bad Gateway                Upstream service unavailable
  503      Service Unavailable        Server maintenance/overload


J.5 GLOSSARY OF VALIDATION RULES

  Field          Min Len  Max Len  Format / Allowed Chars
  -------------  -------  -------  ----------------------------------

                                   @, ., _, -, +, %
